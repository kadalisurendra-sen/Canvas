# US-023: Audit Log Listing with Filters

**As an** admin
**I want to** view a filterable, paginated audit log of all admin actions
**So that** I can track who did what and when for compliance and troubleshooting

## Acceptance Criteria

- [ ] AC1: Audit Log tab on the Analytics page displays filter controls: User dropdown (lists all tenant users), Action Type dropdown (Login/Logout, Configuration Change, Data Export), Date Range picker (from/to date inputs)
- [ ] AC2: "Apply Filters" button applies selected filters; "Clear" button resets all filters
- [ ] AC3: `GET /api/v1/analytics/audit-logs` accepts query params: `user_id`, `action`, `event_type`, `from_date`, `to_date`, `page`, `page_size`
- [ ] AC4: Response returns paginated audit log entries with: id, created_at, user_name, event_type, action, details, ip_address
- [ ] AC5: Log table columns: Date/Time (formatted locale string), Actor (user name), Event (badge: MANAGEMENT=blue, SYSTEM=gray, SECURITY=orange), Details (truncated with expand), IP Address
- [ ] AC6: Event type badges are color-coded: MANAGEMENT (blue), SYSTEM (gray), SECURITY (orange/warning)
- [ ] AC7: Pagination at bottom shows "Showing X of Y results" with page navigation
- [ ] AC8: Table is sorted by created_at descending (newest first) by default
- [ ] AC9: Clicking a row expands to show full details JSON in a formatted view

## Technical Notes
- Backend queries the `audit_logs` table with the index on `created_at DESC` for performance
- User dropdown populated from a separate API call or from Keycloak users list
- Details column stores JSON; frontend parses and displays key-value pairs in the expanded view
