# US-012: Template Wizard Step 5 — Preview & Publish

**As an** admin
**I want to** preview the complete template and publish it or save it as a draft
**So that** I can review everything before making the template available to evaluators

## Acceptance Criteria

- [ ] AC1: Step 5 displays a desktop preview of the full template rendering all stages, sections, and fields as they would appear to an evaluator
- [ ] AC2: A summary stats panel shows: number of stages configured, total fields count, scoring model summary (weighted/unweighted), current status (Draft/Published)
- [ ] AC3: "Publish" button calls `POST /api/v1/templates/{template_id}/publish`, which sets status to "published" and increments the version number
- [ ] AC4: "Save as Draft" button saves the current state without publishing and redirects to the template list
- [ ] AC5: Publish endpoint validates completeness: at least one stage, at least one field per stage, stage weights sum to 100% if scoring is enabled; returns HTTP 422 with specific validation errors if incomplete
- [ ] AC6: On successful publish, user is redirected to the template list with a success toast "Template published successfully"
- [ ] AC7: "Back" returns to Step 4 without losing data
- [ ] AC8: If the template is already published, the preview shows a "Re-publish" button that increments the version

## Technical Notes
- Preview is a read-only rendering of the template structure; no interactivity needed in the preview fields
- Publish is an atomic operation: validate + update status + increment version in a transaction
- Version history is tracked by the integer version column; each publish bumps it by 1
