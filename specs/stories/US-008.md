# US-008: Template Wizard Step 1 — General Info

**As an** admin
**I want to** enter a template's basic information (name, category, description, icon, color, tags)
**So that** I can start creating an evaluation template with its identity and metadata

## Acceptance Criteria

- [ ] AC1: Clicking "Create New Template" navigates to `/templates/new` showing a wizard with a step indicator (Steps 1-5), with Step 1 active
- [ ] AC2: Step 1 form contains: Template Name (required, text input), Category/Domain (required, dropdown: AI/ML Solutions, RPA Automation, Cloud Migration, FinOps Optimization), Description (textarea, 500 char max with character counter), Icon Picker, Theme Color Picker, Search Tags (multi-input, comma-separated or tag chips)
- [ ] AC3: `POST /api/v1/templates` accepts the Step 1 payload and creates a template record with status "draft", returning the template ID
- [ ] AC4: Form validation: name is required and non-empty, category is required, description max 500 chars; inline error messages shown on invalid fields
- [ ] AC5: "Next" button saves the data via API and advances to Step 2; disabled if required fields are empty
- [ ] AC6: "Save as Draft" button saves and returns to the template list
- [ ] AC7: When editing an existing template, the form is pre-populated with current values from `GET /api/v1/templates/{template_id}`
- [ ] AC8: Tags are saved to the `template_tags` table via the template creation/update endpoint

## Technical Notes
- Wizard state managed via React context or URL params (template_id in URL after creation)
- Icon picker can be a simple grid of common Material icon names
- Color picker uses a small palette of predefined colors plus a custom hex input
- Backend creates the template row and associated tags in a single transaction
