# US-016: Tenant Settings — Defaults Tab

**As a** super admin
**I want to** configure default values for my tenant
**So that** new templates and evaluations start with sensible pre-configured settings

## Acceptance Criteria

- [ ] AC1: Defaults tab displays configurable default values: Default evaluation stages (comma-separated list), Default scoring model (weighted/unweighted toggle), Default fail action (Warn/Block/Allow dropdown), Default category for new templates (dropdown)
- [ ] AC2: Form is pre-populated from current tenant configuration via `GET /api/v1/tenants/{tenant_id}`
- [ ] AC3: `PUT /api/v1/tenants/{tenant_id}` accepts updated default settings (stored as JSON in a `defaults` column or as individual columns)
- [ ] AC4: "Save Changes" persists the defaults; "Discard" reverts to last saved values
- [ ] AC5: "Reset to Defaults" restores system-level default values with a confirmation prompt
- [ ] AC6: Changes to defaults only affect newly created templates; existing templates are not modified

## Technical Notes
- Default values can be stored as a JSONB column on the tenants table or as a separate tenant_defaults table
- The template wizard (US-008, US-009) reads these defaults to pre-populate its forms
- Validation ensures all values are within allowed enums
