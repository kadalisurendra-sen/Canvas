# US-019: Master Data Reorder Values

**As an** admin
**I want to** reorder master data values via drag-and-drop
**So that** the display order of reference data matches our organizational preferences

## Acceptance Criteria

- [ ] AC1: Each row in the master data values table has a drag handle (grip icon) on the left side
- [ ] AC2: Rows can be reordered via drag-and-drop; visual feedback shows the drop target position
- [ ] AC3: After dropping a row, `PUT /api/v1/master-data/categories/{cat_id}/reorder` is called with `{ value_ids: [ordered list of UUIDs] }`
- [ ] AC4: Backend updates the `sort_order` column for all values in the category to match the new order
- [ ] AC5: Optimistic UI update: the table reorders immediately; on API failure, the order reverts and an error toast is shown
- [ ] AC6: Reordering is disabled when search filter is active (drag handles are hidden)
- [ ] AC7: Reordering works correctly with pagination (reorders within the full dataset, not just the visible page)

## Technical Notes
- Use the same drag-and-drop library as the template stage reorder (US-009) for consistency
- Backend receives the full ordered list of value IDs and bulk-updates sort_order in a single transaction
- If pagination is active, consider sending only the changed items or fetching the full list for reorder
