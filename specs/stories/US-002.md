# US-002: Keycloak Integration & JWT Validation Middleware

**As a** platform administrator
**I want to** authenticate via Keycloak OIDC and have my JWT validated on every API request
**So that** only authenticated users can access the admin panel APIs

## Acceptance Criteria

- [ ] AC1: FastAPI middleware validates JWT Bearer tokens on all `/api/v1/*` endpoints (except `/api/v1/health` and `/api/v1/auth/*`)
- [ ] AC2: JWT validation fetches the JWKS from the Keycloak realm endpoint and caches the keys
- [ ] AC3: `POST /api/v1/auth/login` initiates the Keycloak OIDC authorization code flow with PKCE
- [ ] AC4: `POST /api/v1/auth/callback` exchanges the authorization code for tokens, validates the ID token, and returns the access token in an httpOnly cookie
- [ ] AC5: `POST /api/v1/auth/logout` revokes the token at Keycloak and clears the session cookie
- [ ] AC6: `GET /api/v1/auth/me` returns the current user profile (user_id, email, name, roles, tenant_id) extracted from the JWT claims
- [ ] AC7: Tenant ID is resolved from the JWT `tenant_id` claim and used to route to the correct tenant database
- [ ] AC8: Requests with expired, malformed, or missing tokens return HTTP 401 with a JSON error body

## Technical Notes
- Use python-jose or PyJWT for JWT decoding and validation
- Keycloak JWKS endpoint: `{KEYCLOAK_URL}/realms/{realm}/.well-known/openid-configuration`
- Store JWKS keys in Redis cache with a 1-hour TTL
- FastAPI dependency `get_current_user` extracts and validates the JWT, returning a typed user context object
- RBAC roles: System Admin, Admin, Contributor, Viewer — stored in JWT realm_access.roles
