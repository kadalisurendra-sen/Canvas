# US-018: Master Data CRUD — List, Add, Edit, Toggle Values

**As an** admin
**I want to** view, add, edit, and toggle the active status of master data values within a category
**So that** I can maintain accurate reference data used across templates and evaluations

## Acceptance Criteria

- [ ] AC1: `GET /api/v1/master-data/categories/{cat_id}/values` returns paginated values with fields: id, value, label, severity, description, is_active, sort_order
- [ ] AC2: API supports `search` (label/value substring), `page`, `page_size` query parameters
- [ ] AC3: Main content area shows a data table with columns: Value, Label, Severity, Description, Status (toggle), Actions (edit/delete)
- [ ] AC4: "Add Value" button opens an inline form row at the top of the table with inputs for: value, label, severity (dropdown: high/medium/low/none), description
- [ ] AC5: `POST /api/v1/master-data/categories/{cat_id}/values` creates a new value; on success the table refreshes
- [ ] AC6: Clicking "Edit" on a row makes the row editable inline; `PUT /api/v1/master-data/values/{value_id}` saves changes
- [ ] AC7: Status toggle switches `is_active` between true/false with immediate API call and optimistic UI update
- [ ] AC8: `DELETE /api/v1/master-data/values/{value_id}` removes the value with a confirmation modal
- [ ] AC9: Search input filters values within the current category (debounced 300ms)
- [ ] AC10: Pagination controls at the bottom show "Showing X of Y values"

## Technical Notes
- Inline editing uses controlled inputs that switch between display and edit mode
- Optimistic updates for toggle with rollback on API failure
- Severity is optional; null/empty severity is valid for categories that don't use it
