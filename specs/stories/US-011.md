# US-011: Template Wizard Step 4 — Scoring & Formulas

**As an** admin
**I want to** configure stage weights, qualification thresholds, and fail actions
**So that** the evaluation scoring model is well-defined and enforceable

## Acceptance Criteria

- [ ] AC1: Step 4 displays a card per stage showing: stage name, weight percentage input, min pass score input, fail action dropdown (Warn/Block/Allow)
- [ ] AC2: Stage weight percentages are displayed as numeric inputs that must sum to 100%; a validation error is shown if they don't
- [ ] AC3: A distribution chart (bar or donut) visually shows the weight distribution across stages, updating in real-time as values change
- [ ] AC4: Min pass score is a numeric input (0-100); when left empty, there is no qualification threshold for that stage
- [ ] AC5: Fail action dropdown defaults to "Warn"; options are Warn (advisory), Block (prevents progression), Allow (ignore threshold)
- [ ] AC6: `PUT /api/v1/templates/{template_id}/scoring` accepts `[{ stage_id, weight_pct, min_pass_score, fail_action }]` and updates the template_stages table
- [ ] AC7: "Next" saves scoring config and advances to Step 5; "Back" returns to Step 3
- [ ] AC8: A summary section shows: total stages, total scoring fields count, scoring model description

## Technical Notes
- Use a simple chart library (e.g., recharts or chart.js) for the weight distribution visualization
- Backend validates that weight_pct values across all stages sum to exactly 100.00
- Fail action is stored as an enum string in the template_stages table

## Dependencies
- US-009 (Wizard Step 2 — stages data)
- US-010 (Wizard Step 3 — fields with scoring flags)
