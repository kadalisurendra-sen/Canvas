# US-013: Delete Template

**As an** admin
**I want to** delete an evaluation template
**So that** unused or erroneous templates are removed from the system

## Acceptance Criteria

- [ ] AC1: Each template card (in the list view) has a "Delete" action button (icon button, ghost style)
- [ ] AC2: Clicking "Delete" opens a confirmation modal: "Are you sure you want to delete '{template name}'? This action cannot be undone."
- [ ] AC3: Modal has "Delete" (red/destructive style) and "Cancel" buttons
- [ ] AC4: `DELETE /api/v1/templates/{template_id}` deletes the template and all associated data (stages, sections, fields, options, tags) via cascade
- [ ] AC5: On successful deletion, the modal closes, a success toast is shown, and the template list refreshes
- [ ] AC6: If the template is published and in use, the API returns HTTP 409 with an error message; the modal shows this error
- [ ] AC7: Only Admin or System Admin roles can delete templates

## Technical Notes
- Database cascading deletes handle child records (template_stages -> template_sections -> template_fields -> field_options, and template_tags)
- Consider soft-delete (setting status to "archived") for published templates instead of hard delete
- Audit log entry created for template deletion
