# US-007: List Evaluation Templates

**As an** admin
**I want to** see all evaluation templates in a list/grid view with status indicators
**So that** I can manage and organize my tenant's evaluation templates

## Acceptance Criteria

- [ ] AC1: `GET /api/v1/templates` returns a list of templates with fields: id, name, category, description, icon, theme_color, status, version, created_by, created_at, updated_at
- [ ] AC2: API supports query parameters: `status` (draft/published/archived), `category`, `search` (name substring), `page`, `page_size`
- [ ] AC3: UI renders template cards in a grid layout showing: template name, category badge, description preview (truncated), status badge, version number, last updated date
- [ ] AC4: Status badges use colors: purple outline for "Draft", green for "Published", gray for "Archived"
- [ ] AC5: "Create New Template" button (primary purple) is displayed at the top right
- [ ] AC6: Each template card has "Edit" and "Delete" action buttons
- [ ] AC7: Clicking a template card or "Edit" navigates to the template wizard (US-008)
- [ ] AC8: Search input and category/status filter dropdowns are displayed above the grid

## Technical Notes
- Backend queries the tenant database `templates` table with joins to `template_tags` for tag data
- React Query with pagination; cards use the design system (white bg, rounded-xl, shadow-sm)
- Template icon uses a Material icon rendered in the card header with the template's theme_color
