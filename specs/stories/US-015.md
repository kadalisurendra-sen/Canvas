# US-015: Tenant Settings — Branding Tab

**As a** super admin
**I want to** customize my tenant's branding (colors, favicon, font, email signature)
**So that** the platform matches my organization's visual identity

## Acceptance Criteria

- [ ] AC1: Branding tab displays form fields: Primary Brand Color (color picker with hex input, default #5F2CFF), Favicon Upload (SVG/PNG/ICO, max 512x512, with preview), Font Family dropdown (Montserrat/Inter/Roboto/Open Sans), Email Signature (textarea)
- [ ] AC2: Color picker shows a visual swatch and accepts manual hex input (validated as 6-char hex)
- [ ] AC3: Favicon upload validates format (SVG/PNG/ICO) and dimensions (max 512x512); shows preview of uploaded favicon
- [ ] AC4: `PUT /api/v1/tenants/{tenant_id}/branding` accepts `{ primary_color, favicon_url, font_family, email_signature }` and updates the tenants table
- [ ] AC5: "Save Changes" persists changes; "Discard" reverts to last saved values
- [ ] AC6: "Reset to Defaults" button restores branding values to system defaults (primary_color=#5F2CFF, font_family=Montserrat, no favicon, empty signature) with a confirmation prompt
- [ ] AC7: Live preview section shows how a sample card/button looks with the selected branding

## Technical Notes
- Favicon file uploaded similarly to logo; stored URL in `tenants.favicon_url`
- Font family selection updates the CSS custom property at the tenant level
- Branding endpoint is separate from general settings to allow partial updates
