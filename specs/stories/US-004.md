# US-004: List Users with Search, Filter & Pagination

**As an** admin
**I want to** view a paginated table of all users in my tenant with search and filter capabilities
**So that** I can find and manage users efficiently

## Acceptance Criteria

- [ ] AC1: `GET /api/v1/users` returns a paginated list of users with fields: id, name, email, role, status, last_login
- [ ] AC2: API supports query parameters: `search` (name/email substring), `role` (Admin/Contributor/Viewer), `status` (Active/Invited/Deactivated), `page`, `page_size`
- [ ] AC3: Response includes pagination metadata: `total`, `page`, `page_size`, `total_pages`
- [ ] AC4: UI renders a data table with columns: Name, Email, Role, Status, Last Login, Actions
- [ ] AC5: Status column displays colored badges: green for Active, yellow for Invited, red for Deactivated
- [ ] AC6: Search input filters users by name or email as the user types (debounced 300ms)
- [ ] AC7: Role and Status dropdown filters update the table when changed
- [ ] AC8: Pagination controls at the bottom show "Showing X of Y users" and page navigation
- [ ] AC9: Users are fetched from Keycloak realm users API (not stored locally), scoped to the current tenant's realm

## Technical Notes
- Backend calls Keycloak Admin REST API to list realm users, merging with any local metadata
- Use React Query for data fetching with automatic refetch on filter/search changes
- Table uses striped rows, sticky header, and hover highlight per the design system
- Ensure tenant isolation: only users from the current tenant's Keycloak realm are returned
