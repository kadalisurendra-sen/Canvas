# US-025: Shared Layout — Sidebar Navigation, Header & Routing

**As a** user
**I want to** navigate between all sections of the admin panel via a sidebar and see consistent header/layout
**So that** I can efficiently move between features with a clear sense of where I am

## Acceptance Criteria

- [ ] AC1: Main layout has a fixed left sidebar (240px width), a top header bar, and a scrollable main content area
- [ ] AC2: Sidebar has dark navy background (#1E2345) with "Helio Canvas 2.0" branding and logo at the top
- [ ] AC3: Navigation items with Material icons: Template Management, User Management, Tenant Settings, Analytics & Audit Logs, Master Data Management
- [ ] AC4: Active navigation item is highlighted with an accent color left border and background tint; sidebar text is muted gray (#A0AEC0), white when active
- [ ] AC5: Sidebar is collapsible: clicking a toggle button collapses it to 64px (icons only); state is persisted in localStorage
- [ ] AC6: Bottom of sidebar shows user profile section: avatar, name, role badge (using role from JWT)
- [ ] AC7: Top header bar shows: breadcrumb/page title (left), notification bell icon (right), user avatar dropdown with Profile and Logout options (right)
- [ ] AC8: React Router v6 routes are configured: `/templates`, `/users`, `/settings`, `/analytics`, `/master-data`, `/login`
- [ ] AC9: Unauthenticated users are redirected to `/login`; authenticated users accessing `/login` are redirected to `/templates`
- [ ] AC10: Font is Montserrat throughout the application; content area background is #F7F8FC

## Technical Notes
- Layout component wraps all authenticated routes; Login page uses a separate layout without the sidebar
- Sidebar collapse state stored in localStorage and read on mount
- User context (name, role, avatar) provided via React Context from the JWT/auth state
- React Router `<Outlet>` pattern for nested routing within the layout
- Notification bell is a placeholder for now (no notification system in scope)

## Dependencies
- US-002 (Authentication — JWT for user context and route protection)
