# US-014: Tenant Settings — General Tab

**As a** super admin
**I want to** configure my tenant's general settings (name, logo, timezone, language, default template)
**So that** the platform reflects my organization's identity and preferences

## Acceptance Criteria

- [ ] AC1: Tenant Settings page at `/settings` displays three tabs: General, Branding, Defaults; General tab is active by default
- [ ] AC2: General tab form contains: Organization Name (text input, pre-filled), Logo Upload (drag-and-drop area accepting PNG/SVG, max 2MB, with preview), Timezone dropdown (Eastern/Pacific/London/UTC), Default Language dropdown (English/Spanish/French), Default Template dropdown (lists published templates)
- [ ] AC3: `GET /api/v1/tenants/{tenant_id}` returns current tenant settings to populate the form
- [ ] AC4: `PUT /api/v1/tenants/{tenant_id}` accepts updated general settings and persists them to the tenants table
- [ ] AC5: Logo upload validates file type (PNG/SVG only) and size (max 2MB); shows error for invalid files
- [ ] AC6: "Save Changes" button saves the form; "Discard" reverts to the last saved state
- [ ] AC7: "Back to All Tenants" link is visible for System Admin role, navigating to tenant list
- [ ] AC8: Success toast shown on save; validation errors shown inline

## Technical Notes
- Logo file is uploaded to a local file storage path (or S3 in production); the URL is stored in `tenants.logo_url`
- Tenant ID is extracted from the JWT claims; the user can only edit their own tenant (unless System Admin)
- Form state managed with React Hook Form or similar; dirty detection for the Discard button
