# US-001: Project Scaffolding & Platform Database

**As a** developer
**I want to** have a working FastAPI + React project with Docker Compose and the platform database schema
**So that** all subsequent features have a solid foundation to build upon

## Acceptance Criteria

- [ ] AC1: FastAPI application starts on port 8000 with a health check endpoint `GET /api/v1/health` returning `{"status": "ok"}`
- [ ] AC2: React (Vite) application starts on port 5173 with a placeholder landing page
- [ ] AC3: Docker Compose file orchestrates PostgreSQL, Redis, FastAPI, and React services
- [ ] AC4: Platform database `db_platform` is created with `tenants` and `tenant_plans` tables matching the schema spec (all columns, types, constraints)
- [ ] AC5: Alembic is configured for platform database migrations; initial migration creates the tenants and tenant_plans tables
- [ ] AC6: SQLAlchemy 2.0 async models exist for `tenants` and `tenant_plans` in the Repo layer (`src/repo/`)
- [ ] AC7: Configuration module in `src/config/` loads DATABASE_URL, REDIS_URL, and KEYCLOAK_URL from environment variables (no hardcoded secrets)

## Technical Notes
- Use asyncpg as the PostgreSQL driver for async SQLAlchemy
- Redis connection via aioredis for caching tenant DB connection strings
- Project structure must follow the 6-layer architecture: Types(0) -> Config(1) -> Repo(2) -> Service(3) -> Runtime(4) -> UI(5)
- Pydantic settings for configuration management

## Dependencies
- None (this is the foundation story)
