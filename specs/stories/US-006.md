# US-006: Edit User Role & Deactivate User

**As an** admin
**I want to** change a user's role or deactivate their account
**So that** I can manage access levels and remove users who no longer need access

## Acceptance Criteria

- [ ] AC1: Each user row in the table has an Actions column with "Edit" and "Deactivate" icon buttons
- [ ] AC2: Clicking "Edit" opens an inline edit mode or modal with a role dropdown (Admin, Contributor, Viewer) pre-populated with the current role
- [ ] AC3: `PUT /api/v1/users/{user_id}` accepts `{ role, status }` and updates the user in Keycloak
- [ ] AC4: On successful role change, a success toast is shown and the table row updates immediately
- [ ] AC5: Clicking "Deactivate" opens a confirmation modal: "Are you sure you want to deactivate {user name}? They will lose access immediately."
- [ ] AC6: `DELETE /api/v1/users/{user_id}` sets the user's status to Deactivated in Keycloak (disables the account, does not delete)
- [ ] AC7: Deactivated users appear in the list with a red "Deactivated" badge and cannot sign in
- [ ] AC8: Admins cannot deactivate themselves
- [ ] AC9: Only Admin or System Admin roles can perform edit and deactivate actions

## Technical Notes
- Backend uses Keycloak Admin REST API to update user attributes and enabled status
- Deactivation sets `enabled: false` on the Keycloak user; it does not delete the user
- Audit log entries are created for both role change and deactivation actions
- Optimistic UI update on role change for responsiveness
