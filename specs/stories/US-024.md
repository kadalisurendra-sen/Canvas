# US-024: Analytics & Audit Log Export

**As an** admin
**I want to** export analytics reports and audit logs as downloadable files
**So that** I can share data with stakeholders and maintain offline records

## Acceptance Criteria

- [ ] AC1: "Export Report" button on the Dashboard tab triggers a download of the analytics report
- [ ] AC2: `GET /api/v1/analytics/export?from={date}&to={date}&format=csv` returns a CSV file with dashboard metrics and chart data tables
- [ ] AC3: Export includes sections: KPI Summary, Stage Distribution, Template Usage, Evaluations Timeline, Top Users
- [ ] AC4: "Export Log" button on the Audit Log tab triggers a download of the filtered audit log
- [ ] AC5: Audit log export respects the currently applied filters (user, action type, date range)
- [ ] AC6: Export format is CSV with columns: Date/Time, Actor, Event Type, Action, Details, IP Address
- [ ] AC7: Both export buttons show a loading spinner while the file is being generated
- [ ] AC8: Downloaded files are named with a timestamp: `analytics_report_YYYYMMDD.csv`, `audit_log_YYYYMMDD.csv`
- [ ] AC9: An audit log entry is created for each export action (event_type: MANAGEMENT, action: data_export)

## Technical Notes
- Backend generates CSV using Python's csv module and returns it as a streaming response with `Content-Disposition: attachment` header
- For large audit logs, use streaming response to avoid memory issues
- File download triggered via a temporary URL or direct blob download in the browser
