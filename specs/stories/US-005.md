# US-005: Invite User via Email

**As an** admin
**I want to** invite a new user by email and assign them a role
**So that** new team members can access the tenant's admin panel

## Acceptance Criteria

- [ ] AC1: "Invite User" button on the User Management page opens a modal dialog
- [ ] AC2: Modal contains email input (validated as valid email format), role dropdown (Admin, Contributor, Viewer), and Send Invite / Cancel buttons
- [ ] AC3: `POST /api/v1/users/invite` accepts `{ email, role }` and creates the user in the tenant's Keycloak realm with status "Invited"
- [ ] AC4: Keycloak sends an invitation email with a link to set up the account
- [ ] AC5: On successful invite, the modal closes, a success toast is shown, and the user list refreshes showing the new user with "Invited" badge
- [ ] AC6: If the email already exists in the realm, the API returns HTTP 409 and the modal shows "User with this email already exists"
- [ ] AC7: Send Invite button is disabled while the request is in progress (loading spinner)
- [ ] AC8: Only users with Admin or System Admin role can see and use the Invite User button

## Technical Notes
- Backend uses Keycloak Admin REST API to create user and trigger the "execute actions" email (VERIFY_EMAIL + UPDATE_PASSWORD)
- An audit log entry is created for the invite action
- Role is assigned via Keycloak realm role mapping
