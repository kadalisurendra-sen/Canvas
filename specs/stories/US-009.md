# US-009: Template Wizard Step 2 — Stage Configuration

**As an** admin
**I want to** define and reorder evaluation stages for a template
**So that** evaluations follow a structured multi-stage process (e.g., Desirable, Feasible, Viable)

## Acceptance Criteria

- [ ] AC1: Step 2 displays a list of stages with an "Add Stage" button
- [ ] AC2: Each stage row shows: stage name (editable inline), sort order number, drag handle for reordering, and a remove (X) button
- [ ] AC3: Stages can be reordered via drag-and-drop; sort_order updates accordingly
- [ ] AC4: Default stages are pre-populated for new templates: "Desirable", "Feasible", "Viable" (user can modify or remove them)
- [ ] AC5: "Add Stage" appends a new empty stage row at the bottom
- [ ] AC6: At least one stage is required; removing the last stage shows a validation error
- [ ] AC7: `PUT /api/v1/templates/{template_id}/stages` accepts an array of stages `[{ name, sort_order }]` and replaces all existing stages for the template
- [ ] AC8: "Next" button saves stages via API and advances to Step 3; "Back" returns to Step 1
- [ ] AC9: Stage names must be non-empty and unique within the template

## Technical Notes
- Use a drag-and-drop library (e.g., @dnd-kit/core or react-beautiful-dnd) for reordering
- Backend deletes existing stages and re-inserts in a transaction (cascade will handle sections/fields cleanup)
- Stage IDs are returned in the response so subsequent steps can reference them
