# US-020: Master Data CSV Import

**As an** admin
**I want to** import master data values from a CSV file
**So that** I can bulk-load reference data without manual entry

## Acceptance Criteria

- [ ] AC1: "Import from CSV" button on the master data page opens an import modal
- [ ] AC2: Modal contains a file upload area (drag-and-drop or click to browse) accepting only .csv files
- [ ] AC3: After file selection, a preview table shows the first 5 rows of parsed CSV data with column mapping: value, label, severity, description
- [ ] AC4: CSV must have a header row; columns are matched by name (case-insensitive); unrecognized columns are ignored
- [ ] AC5: "Import" button calls `POST /api/v1/master-data/categories/{cat_id}/import` with the CSV file as multipart form data
- [ ] AC6: Backend parses the CSV, validates each row (value and label required, severity must be valid enum or empty), and inserts valid rows
- [ ] AC7: Response includes: `{ imported: N, skipped: M, errors: [{ row: X, message: "..." }] }`
- [ ] AC8: Modal shows import results summary; on close, the values table refreshes
- [ ] AC9: Duplicate values (same `value` within the category) are skipped, not treated as errors
- [ ] AC10: Maximum 1000 rows per import; files exceeding this show an error before upload

## Technical Notes
- Use Python's csv module for parsing; handle common encoding issues (UTF-8, UTF-8 BOM)
- Bulk insert via SQLAlchemy `insert().values([...])` for performance
- New values are appended at the end of the sort order (max sort_order + 1, +2, etc.)
- Audit log entry created for the import action with count of imported rows
